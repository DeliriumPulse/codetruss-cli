# Contributing

Thanks for helping improve CodeTruss.

## Issues and product feedback

Public issues are welcome for reproducible bugs, documentation gaps, and focused
feature proposals. Search existing issues first and include the CLI version,
operating system, Git version, expected result, actual result, and the smallest
safe reproduction you can provide.

Never attach private repository source, unredacted diffs or receipts,
credentials, signing keys, provider keys, `.env` files, or customer data.

## Security reports

Do not open a public issue for a suspected vulnerability. Follow
[SECURITY.md](SECURITY.md) instead.

## Code contributions

The CodeTruss-authored source in this repository is proprietary. Unsolicited
pull requests are not accepted until CodeTruss publishes contribution terms
that are compatible with that license. Please open an issue before preparing a
code change. Maintainers will close uncoordinated source changes without merging
them.

This policy avoids asking contributors to create derivative work without clear
licensing terms. It may be revised if CodeTruss adopts a contributor agreement
or a different source license.
